🚫 NOT USED — Forbidden Frameworks
The following are explicitly not part of this project and must never be introduced:
  • Vue / Vue 3 / Nuxt
  • Svelte / SvelteKit
  • Angular
  • Livewire-only UI
  • Blade-only UI (Blade is only used as the Inertia root view)
  • MySQL or SQLite as a database

No Vue adapters anywhere in the codebase.

§7 Spatie RBAC (laravel-permission)

Installation

# 1. Require package
composer require spatie/laravel-permission

# 2. Publish config & migration
php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider"

# 3. Run migrations (creates 5 Spatie tables automatically)
php artisan migrate

# 4. Seed roles & permissions
php artisan db:seed --class=RolesAndPermissionsSeeder
⚠️ Do NOT manually create Spatie tables — they are auto-created by php artisan migrate.

§9 Acceptance Verification Queries

Run these PostgreSQL queries after migration to verify correctness.

V1 · UUID PK + No assigned_to_user_id

SELECT column_name, data_type, column_default
FROM information_schema.columns
WHERE table_name = 'tasks'
  AND column_name IN ('id','assigned_to_user_id','status','priority','due_at');
-- Expected: id=uuid, no assigned_to_user_id row, due_at=timestamp with time zone

V2 · Enum CHECK Constraints

SELECT con.conname, pg_get_constraintdef(con.oid)
FROM pg_constraint con
JOIN pg_class rel ON rel.oid = con.conrelid
WHERE rel.relname = 'tasks' AND con.contype = 'c';
-- Expected: chk_task_status, chk_task_priority

V3 · Polymorphic VARCHAR(64)

SELECT table_name, column_name, character_maximum_length
FROM information_schema.columns
WHERE column_name IN ('commentable_id','linkable_id','entity_id','subject_id')
  AND table_schema = 'public';
-- Expected: all 4 rows with character_maximum_length = 64

V4 · Required Indexes (expect 6+ rows)

SELECT indexname, tablename FROM pg_indexes
WHERE indexname IN (
  'idx_ct_poly','idx_tl_linkable','idx_notif_entity',
  'idx_tr_task','idx_tr_remind_at','idx_np_user'
);
-- Expected: 6 rows

V5 · Unique on notification_preferences

SELECT con.conname FROM pg_constraint con
JOIN pg_class rel ON rel.oid = con.conrelid
WHERE rel.relname = 'notification_preferences' AND con.contype = 'u';
-- Expected: uq_np_user_type_channel

V6 · Zero tenant_id (expect 0 rows)

SELECT table_name, column_name FROM information_schema.columns
WHERE column_name = 'tenant_id' AND table_schema = 'public';
-- Expected: 0 rows

V7 · notifications.id is BIGSERIAL

SELECT column_name, data_type FROM information_schema.columns
WHERE table_name = 'notifications' AND column_name = 'id';
-- Expected: data_type = bigint
